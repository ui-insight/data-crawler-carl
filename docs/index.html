How to get a free API key

  1. Visit Google AI Studio and sign in with your Google account.
  2. Click Create API Key and select a Google Cloud project (or let it create one).
  3. Copy the generated key and paste it into the field above.

Your key is stored only in your browser's session memory — it is never saved to disk or sent anywhere except directly to Google's API. It is cleared when you close the tab.

How Data Crawler Carl Works

The AI Model

Carl is powered by Gemini 2.5 Flash, a large language model (LLM) from Google. LLMs work by predicting the most likely next token (word/symbol) given the conversation so far — they are sophisticated pattern-matching systems trained on vast amounts of text.

How a Query Works

1. CSV → SQLite — When you upload a CSV, it is parsed with PapaParse and loaded into an in-browser SQLite database powered by sql.js (SQLite compiled to WebAssembly). No server, no backend — the full dataset never leaves your browser.

2. Round 1 — AI writes SQL — Your question is sent to Gemini along with only the column names and a 5-row sample. Gemini responds with SQL queries to answer your question.

3. Local execution — The SQL runs entirely in your browser against the local SQLite database. Results appear inline in the chat.

4. Round 2 — AI analyzes real data — The actual query results are sent back to Gemini, which now provides analysis, insights, and chart specifications based on the real numbers.

5. Safe Charts — Gemini provides a chart type, a SQL query, and column mappings. The system executes the SQL and renders the chart with Plotly.js using the real data. Supports bar, scatter, line, pie, histogram, box, and heatmap. No eval(), no arbitrary code execution.

Security & Privacy

  • Only column names + 5-row sample are sent to the Gemini API. The full dataset, all query results, and all chart data stay local.
  • API key is held in session memory only — never saved to disk, cleared when you close the tab.
  • Charts are rendered from safe JSON specs — no eval(), no Function(), no arbitrary code.
  • No cookies, no tracking, no analytics.

About Data Crawler Carl

An AI-powered CSV data explorer that runs entirely in your browser. Upload any dataset, ask questions in plain English, and get back SQL queries, charts, and analysis.

What you can do

  • Upload any CSV — drag & drop or browse. Data is parsed and loaded into an in-browser SQLite database.
  • Ask questions in natural language — "What's the average salary by department?" or "Show me the top 10 cities."
  • Auto-generated SQL — The AI writes SQLite queries that run locally against your data. Results appear inline and in the SQL tab.
  • Charts — Bar, scatter, line, pie, histogram, box, and heatmap charts rendered with Plotly.js. Ask for trendlines too.
  • Iterate — Follow up with more questions. The AI remembers context within the conversation.

Privacy & Security

  • Your data stays local. CSV files are parsed and queried entirely in your browser using WebAssembly (sql.js). No data is uploaded to any server.
  • Only the schema is sent to AI. Column names and a small sample (first 5 rows) are sent to Google Gemini to provide context for your questions.
  • API key is session-only. Your Gemini API key is held in session memory and cleared when you close or reload the tab. It is never saved to disk.
  • No code execution. Charts are rendered from safe JSON specs — no eval() or arbitrary code runs.
  • Static site. This is a static GitHub Pages site with no backend, no cookies, no tracking, and no analytics.
