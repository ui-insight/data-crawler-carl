How Data Crawler Carl Works

Architecture

Data Crawler Carl is a static website hosted on GitHub Pages. There is no backend server, no database server, no application server — the entire application is a collection of HTML, CSS, and JavaScript files served as static assets. Everything runs in your browser. The source code is open source on GitHub.

The AI Model

Carl uses Google Gemini 2.5 Flash, a large language model (LLM). LLMs work by predicting the most likely next token (word/symbol) given context — they are sophisticated pattern-matching systems trained on vast amounts of text. The browser communicates directly with the Google Generative AI API using the @google/genai JavaScript SDK (loaded from an ESM CDN). API calls go straight from your browser to Google — no proxy, no middleware.

You provide your own API key, which is stored in sessionStorage (browser session memory). The key is never written to disk, never sent to any server other than Google's API, and is automatically cleared when you close the tab.

In-Browser SQL with WebAssembly

When you upload a CSV, it is parsed with PapaParse (a client-side CSV parser) and loaded into SQLite running entirely in your browser via sql.js — the full SQLite database engine compiled from C to WebAssembly (Wasm). WebAssembly lets native-speed code run in the browser sandbox. Your dataset lives in memory as a real SQLite database, supporting full SQL queries, aggregations, joins, and more — all locally, with zero network calls.

The Two-Round Conversation

Each question you ask triggers a two-round exchange designed to prevent hallucinated data:

Round 1 — AI writes SQL: Your question is sent to Gemini along with only the column names and a 5-row sample of your data. Gemini responds with SQL queries — it never sees or guesses at the full dataset. The SQL is executed locally against your in-browser SQLite database and results appear inline.

Round 2 — AI analyzes real data: The actual query results (real numbers from your database) are sent back to Gemini. Only now does it provide analysis, insights, and chart specifications — grounded in real data, not predictions.

This separation means Gemini never fabricates statistics. It proposes what to compute; your browser computes it; then Gemini interprets the real results.

Safe Chart Rendering

When Gemini wants to create a chart, it returns a declarative JSON spec containing a chart type, a SQL query, and column mappings (e.g. "x maps to Department, y maps to avg_salary"). The system executes the SQL itself, extracts the data, and renders the chart with Plotly.js. Gemini never provides raw data for charts and never generates executable code. There is no eval(), no Function() constructor, no dynamic script injection. Chart types are restricted to an allowlist (bar, scatter, line, pie, histogram, box, heatmap) and all values are sanitized before rendering.

What Gets Sent to Google

  • Column names and a 5-row sample of your CSV (so Gemini understands the schema).
  • Your natural-language questions and the conversation history.
  • Actual query results in Round 2 (so Gemini can analyze them).
  • Your API key (sent directly to Google's API endpoint for authentication).

The full dataset is never uploaded. All SQL execution, all chart data extraction, and all rendering happen locally.

Security Summary

  • No server. Static GitHub Pages site — no backend, no database, no server-side code.
  • No persistent storage. API key in sessionStorage only — cleared on tab close. No cookies, no localStorage.
  • No code execution. Charts rendered from validated JSON specs. Property allowlists per chart type. All strings sanitized (HTML tags stripped).
  • No tracking. No cookies, no analytics, no third-party scripts beyond the Gemini SDK and Plotly.js.

About Data Crawler Carl

An AI-powered CSV data explorer that runs entirely in your browser. Upload any dataset, ask questions in plain English, and get back SQL queries, charts, and analysis.

What you can do

  • Upload any CSV — drag & drop or browse. Data is parsed and loaded into an in-browser SQLite database.
  • Ask questions in natural language — "What's the average salary by department?" or "Show me the top 10 cities."
  • Auto-generated SQL — The AI writes SQLite queries that run locally against your data. Results appear inline and in the SQL tab.
  • Charts — Bar, scatter, line, pie, histogram, box, and heatmap charts rendered with Plotly.js. Ask for trendlines too.
  • Iterate — Follow up with more questions. The AI remembers context within the conversation.

Privacy & Security

  • Your data stays local. CSV files are parsed and queried entirely in your browser using WebAssembly (sql.js). No data is uploaded to any server.
  • Only the schema is sent to AI. Column names and a small sample (first 5 rows) are sent to Google Gemini to provide context for your questions.
  • API key is session-only. Your Gemini API key is held in session memory and cleared when you close or reload the tab. It is never saved to disk.
  • No code execution. Charts are rendered from safe JSON specs — no eval() or arbitrary code runs.
  • Static site. This is a static GitHub Pages site with no backend, no cookies, no tracking, and no analytics.
